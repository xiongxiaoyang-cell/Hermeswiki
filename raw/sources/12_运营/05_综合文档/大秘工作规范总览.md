## 一、知识库SOP

### 核心规则
- 周度政策跟踪 / 重大政策24h内
- 信源≥2个交叉验证
- 8字段标准化模板

### 更新检查（4条）
1. 涉及新国家 → 更新区域政策
2. 涉及新行业 → 更新行业案例
3. 涉及企业案例 → 多库关联
4. **新增文件 → 与库内文件比对 → 记录关联关系**

### 关联记录规则
- 新文件入库时，自动检查与现有文件的关联
- 记录关联类型：引用、补充、替代、扩展
- 在文件元数据或关联记录文件中标注


## 二、任务分发SOP

### 任务来源

| 来源 | 方式 | 处理方式 |
|------|------|----------|
| 手动任务 | 直接说话派发 | 创建记录 → sessions_spawn启动Agent → 执行 → 群通知 → 审核 |
| 定时任务 | Cron自动触发 | 自动分发 → 群聊通知 → 你审核 |

### 执行流程
```
发布任务 → 创建多维表格记录 → sessions_spawn启动子Agent → 执行 → 更新表格 → 发送到对应群聊 → 抄送你审核 → 你审核（OK/通过/需修改）
```

### 协同模式

**并行执行**：任务互不依赖，同时启动多个Agent
```
sessions_spawn(subagent_hr, "政策研究")
sessions_spawn(subagent_ip, "内容策划")
sessions_spawn(subagent_overseas, "市场分析")
```

**串行执行**：下一个任务依赖上一个输出
```
Agent A → 输出结果 → Agent B → 输出结果 → Agent C
```

**混合模式**：并行+串行结合
```
[Agent A, Agent B] 并行 → 汇总 → Agent C 串行
```

### 任务分发判断
- 与用友/薪福社相关 → 通知用友薪福社出海HR/零工群
- 与IP/内容相关 → 通知IP运营群

- 与课程/培训相关 → 通知课程设计群
- 与私域运营相关 → 通知私域运营群

## 五、消息规则

### 审核关键词
| 你说 | 状态更新 |
|------|---------|
| OK/通过/可以 | 已通过 |
| 需修改/不通过 | 需修改 |
| 开始了 | 进行中 |
| 好了/完成了 | 已完成 |

### 消息格式要求
- 简洁清晰
- 重要事项加粗标注
- 使用普通文本（避免卡片）

## 六、多Agent任务协调

### 自动任务分解
收到复杂任务时，大秘自动：
1. 分析任务复杂度
2. 拆解为子任务
3. 识别任务依赖关系
4. 分配给最适合的Agent

### 状态跟踪

| 状态 | 含义 |
|------|------|
| ⏳ 待开始 | 任务已分解，等待执行 |
| 🔄 进行中 | Agent正在执行 |
| ✅ 已完成 | 执行成功，待汇总 |
| ⚠️ 需修改 | 执行结果需调整 |
| 📋 已汇总 | 结果已汇总报告 |
