# 飞书日历同步功能设计

> 创建日期：2026-03-20
> 目标：同步用户日程（读写删）

## 一、权限需求分析

### 1.1 已授权日历权限（24项）

| 权限名称 | 说明 | 同步功能所需 |
|---------|------|-------------|
| `calendar:calendar` | 日历全权限 | ✅ 必需 |
| `calendar:calendar:create` | 创建日历 | ✅ 创建日程 |
| `calendar:calendar:read` | 读取日历 | ✅ 读取日程 |
| `calendar:calendar:update` | 更新日历 | ✅ 更新日程 |
| `calendar:calendar:delete` | 删除日历 | ✅ 删除日程 |
| `calendar:calendar:readonly` | 日历只读 | - |
| `calendar:calendar.event:create` | 创建日历事件 | ✅ 必需 |
| `calendar:calendar.event:read` | 读取日历事件 | ✅ 必需 |
| `calendar:calendar.event:update` | 更新日历事件 | ✅ 必需 |
| `calendar:calendar.event:delete` | 删除日历事件 | ✅ 必需 |
| `calendar:calendar.event:reply` | 回复日历事件 | ❌ 暂不需要 |
| `calendar:calendar.acl:create` | 创建日历权限 | ❌ 暂不需要 |
| `calendar:calendar.acl:read` | 读取日历权限 | ❌ 暂不需要 |
| `calendar:calendar.acl:delete` | 删除日历权限 | ❌ 暂不需要 |
| `calendar:calendar.free_busy:read` | 读取忙闲状态 | ⭐ 可扩展 |
| `calendar:calendar:subscribe` | 订阅日历 | ❌ 暂不需要 |
| `calendar:timeoff` | 休假管理 | ❌ 暂不需要 |
| `calendar:time_off:create` | 创建休假 | ❌ 暂不需要 |
| `calendar:time_off:delete` | 删除休假 | ❌ 暂不需要 |
| `calendar:settings.workhour:read` | 读取工作时间 | ⭐ 可扩展 |
| `calendar:exchange.bindings:*` | Exchange绑定 | ❌ 暂不需要 |
| `calendar:settings.caldav:create` | CALDAV设置 | ❌ 暂不需要 |

**结论**：当前权限完全满足日程同步的读写删需求 ✅

## 二、API 接口清单

### 2.1 日历管理 API

| API | 说明 | 对应权限 |
|-----|------|---------|
| `POST /open-apis/calendar/v4/calendars` | 创建日历 | `calendar:calendar:create` |
| `GET /open-apis/calendar/v4/calendars` | 获取日历列表 | `calendar:calendar:read` |
| `GET /open-apis/calendar/v4/calendars/:calendar_id` | 获取日历详情 | `calendar:calendar:read` |
| `PATCH /open-apis/calendar/v4/calendars/:calendar_id` | 更新日历 | `calendar:calendar:update` |
| `DELETE /open-apis/calendar/v4/calendars/:calendar_id` | 删除日历 | `calendar:calendar:delete` |

### 2.2 日程事件 API（核心）

| API | 说明 | 对应权限 |
|-----|------|---------|
| `POST /open-apis/calendar/v4/calendars/:calendar_id/events` | 创建日程 | `calendar:calendar.event:create` |
| `GET /open-apis/calendar/v4/calendars/:calendar_id/events` | 获取日程列表 | `calendar:calendar.event:read` |
| `GET /open-apis/calendar/v4/calendars/:calendar_id/events/:event_id` | 获取日程详情 | `calendar:calendar.event:read` |
| `PATCH /open-apis/calendar/v4/calendars/:calendar_id/events/:event_id` | 更新日程 | `calendar:calendar.event:update` |
| `DELETE /open-apis/calendar/v4/calendars/:calendar_id/events/:event_id` | 删除日程 | `calendar:calendar.event:delete` |

### 2.3 忙闲查询 API

| API | 说明 | 对应权限 |
|-----|------|---------|
| `POST /open-apis/calendar/v4/freebusy/list` | 批量查询忙闲 | `calendar:calendar.free_busy:read` |

## 三、日程数据结构

### 3.1 Event 对象结构

```json
{
  "event_id": "xxx",
  "summary": "会议标题",
  "description": "会议描述",
  "start": {
    "timezone": "Asia/Shanghai",
    "timestamp": "1704067200"
  },
  "end": {
    "timezone": "Asia/Shanghai",
    "timestamp": "1704070800"
  },
  "location": {
    "name": "会议室A",
    "address": "xxx"
  },
  "attendees": [
    {
      "type": "user",
      "user_id": "ou_xxx",
      "display_name": "张三"
    }
  ],
  "recurrence": "RRULE:FREQ=WEEKLY;BYDAY=MO,WE",
  "status": {
    "is_all_day": false,
    "is_recurring": false
  },
  "visibility": "default",
  "color": "#FF9900"
}
```

### 3.2 时间格式

| 类型 | 格式 | 示例 |
|------|------|------|
| 非全天 | timestamp (秒) + timezone | `"timestamp": "1704067200", "timezone": "Asia/Shanghai"` |
| 全天 | date | `"date": "2024-01-01"` |

## 四、同步功能设计

### 4.1 工具设计

```
feishu_calendar
├── list_calendars        # 列出日历
├── get_calendar         # 获取日历详情
├── create_calendar      # 创建日历
├── update_calendar      # 更新日历
├── delete_calendar      # 删除日历
├── list_events          # 列出日程
├── get_event            # 获取日程详情
├── create_event         # 创建日程
├── update_event         # 更新日程
├── delete_event         # 删除日程
└── check_freebusy       # 查询忙闲状态
```

### 4.2 参数设计

#### 4.2.1 列出日程 (list_events)

```json
{
  "action": "list_events",
  "calendar_id": "primary",  // 或 "ou_xxx:xxx"
  "start_time": "1704067200",  // 可选：起始时间戳
  "end_time": "1704153600",     // 可选：结束时间戳
  "max_results": 50              // 可选：最大数量
}
```

#### 4.2.2 创建日程 (create_event)

```json
{
  "action": "create_event",
  "calendar_id": "primary",
  "summary": "会议标题",
  "description": "会议描述",
  "start_timestamp": "1704067200",
  "end_timestamp": "1704070800",
  "timezone": "Asia/Shanghai",
  "location": "会议室A",
  "attendees": ["ou_xxx", "ou_yyy"],  // 可选：参与者
  "is_all_day": false,
  "color": "#FF9900"
}
```

#### 4.2.3 更新日程 (update_event)

```json
{
  "action": "update_event",
  "calendar_id": "primary",
  "event_id": "event_xxx",
  "summary": "新标题",
  "start_timestamp": "1704070800",
  "end_timestamp": "1704074400"
}
```

#### 4.2.4 删除日程 (delete_event)

```json
{
  "action": "delete_event",
  "calendar_id": "primary",
  "event_id": "event_xxx"
}
```

### 4.3 日历ID说明

| calendar_id | 说明 |
|-------------|------|
| `primary` | 用户的主日历 |
| `ou_xxx:xxx` | 其他用户的日历（需授权） |
| `xxx@group.calendar.feishu.cn` | 群组日历 |

## 五、飞书 SDK 调用示例

### 5.1 客户端初始化

```typescript
import Lark from "@larksuiteoapi/node-sdk";

const client = new Lark.Client({
  appId: process.env.FEISHU_APP_ID,
  appSecret: process.env.FEISHU_APP_SECRET,
});
```

### 5.2 获取日程列表

```typescript
const result = await client.calendar.event.list({
  params: {
    calendar_id: "primary",
    start_time: "1704067200",
    end_time: "1704153600",
  },
});
```

### 5.3 创建日程

```typescript
const result = await client.calendar.event.create({
  path: { calendar_id: "primary" },
  data: {
    summary: "会议标题",
    description: "会议描述",
    start: { timestamp: "1704067200", timezone: "Asia/Shanghai" },
    end: { timestamp: "1704070800", timezone: "Asia/Shanghai" },
  },
});
```

### 5.4 更新日程

```typescript
const result = await client.calendar.event.update({
  path: { calendar_id: "primary", event_id: "event_xxx" },
  data: {
    summary: "新标题",
  },
});
```

### 5.5 删除日程

```typescript
const result = await client.calendar.event.delete({
  path: { calendar_id: "primary", event_id: "event_xxx" },
});
```

## 七、注意事项

### 7.1 权限范围

- 当前授权为 **tenant** 级别，可以访问整个企业内的日历
- 如需访问其他用户日历，需要用户授权 `user` 级别权限

### 7.2 速率限制

- 日历 API 调用频率限制请参考：[飞书API频率限制](https://open.feishu.cn/document/server-docs/api-call-guide/frequency-control)

### 7.3 时区处理

- 飞书内部使用 UTC 存储时间
- API 调用时需指定 `timezone` 参数
- 建议统一使用 `Asia/Shanghai` 时区
