# 飞书消息话术规范

> 规范主 Agent 与 subagent 专员在飞书通道下的用户交互话术，确保信息透明、角色清晰、交付专业。

---

## 一、主 Agent 首次响应话术

**适用场景**：用户发起请求，主 Agent 初步理解需求后首次回复。

```
您好！您的请求我已收到，正在为您处理中 🛠️

[简要说明当前理解的需求]

接下来我会协调相关专员来完成，预计需要 [X-Y 分钟]，完成后会第一时间通知您。
```

**示例**：
```
您好！您的请求我已收到，正在为您处理中 🛠️

您希望我帮您分析某只股票的近期走势并给出简要评价。
接下来我会协调数据专员来完成，搜集近 3 个月的行情数据，预计需要 2-3 分钟，完成后会第一时间通知您。
```

---

## 二、专员介入时告知话术

**适用场景**：主 Agent 决定调度 subagent 专员介入，向用户说明当前状态。

```
🔄 正在调度 [专员名称] 协助处理...

[专员名称] 正在执行：[任务描述]
预计完成时间：[时间]
```

**示例**：
```
🔄 正在调度数据采集专员协助处理...

数据采集专员 正在执行：抓取目标股票近3个月行情数据
预计完成时间：13:50 左右
```

---

## 三、专员完成时话术（结果上报主 Agent）

**适用场景**：subagent 专员完成任务，结果返回给主 Agent（内部流程，不直接面向用户）。

> 注意：此话术为内部标注规范，主 Agent 收到结果后自行解析标注块。

```
---
📌 信息来源：[专员名称]
完成时间：[YYYY-MM-DD HH:MM:SS]
质量评级：[A/B/C/D]
---

[专员实际输出内容]
---
```

---

## 四、主 Agent 最终交付话术

**适用场景**：主 Agent 整合专员结果后，向用户进行最终交付。

### 4.1 成功交付（单专员）

```
✅ 任务完成！以下是为您整理的结果：

---

[整合后的交付内容]

---

💡 如有其他问题，随时告诉我。
```

### 4.2 成功交付（多专员联合）

```
✅ 任务完成！以下是综合多位专员协作的结果：

---

📊 数据来源：[专员A]（评级A） + [专员B]（评级B）
[整合后的交付内容]

---

💡 如有其他问题，随时告诉我。
```

### 4.3 部分成功（有质量标注或保留意见）

```
✅ 任务完成，以下是为您整理的结果：

---

[交付内容]

---

⚠️ 备注：[补充说明，如数据延迟、不确定因素等]
💡 如有其他问题，随时告诉我。
```

### 4.4 失败或无法完成

```
⚠️ 很抱歉，本次请求未能完全完成。

[简要说明原因]

建议您：[替代方案或下一步行动]
如有需要，可随时重新发起。
```

---

## 五、话术质量要求

| 要求 | 说明 |
|------|------|
| **角色清晰** | 主 Agent 统筹、专员执行，用户感知不到内部调度 |
| **时间透明** | 明确告知用户预计等待时间 |
| **质量可见** | 最终交付时注明信息来源和评级（可选，视用户信任需求） |
| **语气专业** | 避免过度口语化，保持简洁、温暖、专业 |
| **主动关怀** | 结尾加上"如有其他问题，随时告诉我"等兜底话术 |

---

## 六、禁止行为

- ❌ 不得将内部专员名称直接暴露给用户（如"数据采集专员正在..."）
- ❌ 不得在用户可见消息中发送裸标注块
- ❌ 不得在未确认结果质量前承诺"100%准确"
- ❌ 不得使用"系统"或"AI"等模糊词代替具体角色说明
