## 一、任务分发规则

### 核心原则
> **任务分配时立即登记看板，不是完成后登记**

### 任务来源

| 来源 | 方式 | 处理方式 |
|------|------|----------|
| 手动任务 | 你直接说话派发 | 创建任务记录（进行中）→ sessions_spawn启动子Agent → 执行 → 更新表格 → 群通知 → 审核 |
| 定时任务 | Cron自动触发 | 自动创建记录 → 自动分发 → 群聊通知 → 你审核 |

---

## 二、完整执行流程

### 流程图

```
你发布任务
    ↓
1. 创建多维表格记录（任务协作看板）← 分配时立即登记
    ↓
2. sessions_spawn 启动子Agent（同步执行多个）
    ↓
3. 子Agent读取配置文件 + 执行任务
    ↓
4. 执行完成 → 更新多维表格状态（进行中→已完成）
    ↓
5. 发送到对应群聊（根据Agent映射）
    ↓
6. 任务关闭
```

### 看板状态与通知机制

| 阶段 | 看板状态 | 自动动作 | 通知 |
|------|---------|---------|------|
| 任务分配 | 进行中 | 创建记录 | - |
| 执行中 | 进行中 | - | - |
| 执行完成 | 已完成 | 更新记录 | 发群 |
| 任务关闭 | 已完成 | 关闭 | - |

### sessions_spawn 使用方法

```bash
# 启动子Agent执行任务
openclaw sessions_spawn \
  --agent-id <agent_id> \
  --task "<任务描述>" \
  --context "<相关背景信息>"
```

## 五、审核流程

### 任务状态流转

```
待处理 → 进行中 → 已完成
              ↓
           待审核 → 已通过（关闭）
              ↓
            需修改 → 进行中
```

### 审核关键词

| 你说 | 我更新状态 |
|--------|-----------|
| OK / 通过 / 可以 | 已通过 |
| 需修改 / 不通过 | 需修改 |
| 进行中 / 开始了 | 进行中 |
| 已完成 / 好了 | 已完成 |

## 六、监察标准

### 需要立即通知的变动

| 变动类型 | 判定条件 | 动作 | 通知范围 |
|----------|----------|------|----------|
| 任务状态变化 | 执行完成 | 群通知 | 相关群 |
| 审核结果 | OK/通过/需修改 | 抄送你 | 个人 |
| 任务超时 | 3分钟未完成 | 重试/通知 | 个人 |

### 定期检查

| 检查项 | 周期 | 动作 |
|--------|------|------|
| 知识库健康度 | 每周五17:00 | 汇报 |
| 任务积压 | 每天 | 检查并通知 |
