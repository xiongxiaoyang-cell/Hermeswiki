## 三、优化建议

### 3.1 目录结构优化

#### 总体原则
- **一层分类维度统一**：按业务域（战略/区域/HR/行业/课程/自媒体/用友/合作/运营）划分，不再交叉
- **二进制文件处置**：制定明确规则（见3.4）
- **重复内容合并**：只保留最新/最完整版本

#### 各目录整改建议

**`01_战略方法论/`**
- 合并3个三问法版本 → 保留一个综合版，命名`三问法学习框架-综合版.md`，其他归档为`三问法学习框架-参考备份/`
- 删除V5整合版（如果与三问法内容重复）

**`02_区域政策/`**
- `国家/`下的各区域汇总（`中东非洲.md`、`欧洲.md`、`美洲.md`、`亚太.md`）→ 合并到`国家/README.md`或删除（`国家/`内已有子区域）
- 保留`CARF/`专项追踪目录
- `README.md`增加区域索引

**`03_人力资源/`**
- `模块/01-战略定位.md` ~ `10-落地执行.md` → 建议重命名为语义标题（如`01_战略定位.md` → `战略定位-中企出海HR方法论.md`），并评估是否与`01_战略方法论/`内容重复
- 整体作为HR专业内容主入口

**`05_课程运营/`**
- 根目录游离文件（`content.md`、`slides.json`、`出海人力资源管理线上课程设计.md`、`第1讲-认知破局.pptx`）→ 移入`出海HR/`子目录
- `迅合大会PPT-52页完整版.pptx` → 保留；`迅合大会PPT.pptx` → 删除
- 考虑增加`课件/`、`学员资料/`、`课程设计/`子目录

**`06_自媒体/`**
- `抖音封面.pptx` → 移入`选题/`或`运营流程/`
- `README.md`保留在根目录作为入口
- `选题/`内的日期文件 → 考虑按月合并（如`2026-03选题汇总.md`）

**`07_用友薪福社/`**
- 知识库架构3个文件 → 合并为1个`知识库架构.md`（V2终稿），其他归档为`知识库架构-历史/`
- 根目录`夏楠专家背书及融合规划.md` → 移入`合作伙伴/`
- 根目录`销售方法论.md` → 移入`服务方案/`或新建`销售工具/`
- `文件提取分析.md` → 评审后决定保留或删除
- `豆包对话记录/` → 评审对话价值，建议移至`研究/`或删除
- `金山科技/HRO成交案例-金山.pdf` → 与`金山科技/HRO服务/金山HRO服务案例.pdf`合并或删除一个

**`08_合作生态/`**
- 结构清晰，保持现状

### 3.2 文件命名规范

#### 命名规则
```
[序号_][主题]-[副标题].[扩展名]
```

**规则说明：**
1. 序号仅在需要排序时使用（`_01`、`_02`……）
2. 主题用中文，准确概括内容
3. 副标题用中文，可省略
4. 日期格式统一为`YYYYMMDD`（无连字符），用于需要标注时效性的文件
5. 避免在文件名中出现"文档"、"文件"等冗余词

#### 命名示例

| 场景 | 规范前 | 规范后 |
|------|--------|--------|
| 日期文件 | `2026-03-14-选题.md` | `20260314-选题.md` |
| 合并版本 | `三问法学习框架-业务出海研究报告.md` | `三问法学习框架-综合版.md` |
| 人物文件 | `David-Yang-Profile.md` | `杨lx-Profile.md` |
| 运营文件 | `知识库架构V2-执行状态.md` | `知识库架构-V2执行状态.md` |

**建议在`00_索引/`增加：**
- `知识库贡献指南.md` — 如何正确新增文件
- `文件命名规范.md` — 本规范详细版
- `待清理文件清单.md` — 列出所有待处理的问题文件

### 3.4 二进制文件处置规则

> 规则：执行文件本地，文档存Gitee仓库

| 类型 | 示例 | 规则 |
|------|------|------|
| 模板文件（.xlsx/.docx） | `交付模板/*.xlsx`、`顾问协议模板.docx` | 本地执行文件，**Gitee仅保留最新版本存档**，确认本地有备份 |
| 演示文件（.pptx） | `迅合大会PPT.pptx` | 若本地有原始文件，Gitee只保留一份最新完整版 |
| 静态资源 | `抖音封面.pptx` | 如无本地备份则保留，否则移出知识库 |
| PDF资料 | `互联网行业出海解决方案-内部分享版.pdf` | 评审是否应转为飞书文档链接 |
