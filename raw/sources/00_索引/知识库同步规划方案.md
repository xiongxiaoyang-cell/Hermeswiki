## 三、实施阶段

### 第一阶段：GitHub ↔ Gitee 双向同步
**目标**：Gitee作为可靠的.md备份库

**实施方案**：
1. **同步策略**：GitHub为主，Gitee自动同步
2. **同步内容**：仅.md文件（二进制走云盘）
3. **同步方式**：定时任务 + Webhook双重保障

**技术实现**：
- 方案A1：使用GitHub Actions定时同步到Gitee
- 方案A2：本地脚本 + 定时执行
- 推荐：方案A1（自动化程度高）

**Gitee同步脚本逻辑**：
```bash
# 1. 克隆GitHub仓库
# 2. 过滤掉二进制文件
# 3. 推送到Gitee
git clone https://github.com/xiongxiaoyang-cell/lobster.git
cd lobster
git filter-branch --prune-empty --subdirectory-filter docs/知识库 HEAD
git push https://gitee.com/stuardyang/lobster.git master
```

- [ ] 创建GitHub Actions workflow
- [ ] 配置Gitee仓库作为推送目标

### 第二阶段：二进制文件归档（55个）
**目标**：规范化管理二进制文件

**现状统计**：

| 分类 | 数量 | 文件类型 |

| 12_运营 | 2 | PPTX, PDF |
| 05_行业案例 | 7 | PDF |

| 07_自媒体 | 1 | PPTX |
| 09_用友薪福社 | 17 | PPTX, PDF |
| **总计** | **33** | |

> 注：另有22个文件分布在其他分类

**实施方案**：
- **小文件（≤10MB）**：使用GitHub LFS存储
- **大文件（>10MB）**：迁移到飞书云盘

**Git LFS配置**：
```bash
# 创建 .gitattributes
*.pptx filter=lfs diff=lfs merge=lfs -text
*.pdf filter=lfs diff=lfs merge=lfs -text
*.docx filter=lfs diff=lfs merge=lfs -text
*.xlsx filter=lfs diff=lfs merge=lfs -text
```

**飞书云盘归档策略**：
1. 按12个分类建立同名文件夹
2. 二进制文件归档到对应文件夹
3. 创建索引文件记录云盘路径

**索引文件格式**（每个分类下创建 `二进制文件索引.md`）：
```markdown
| 文件名 | 大小 | 飞书云盘链接 | 最后更新 |
|--------|------|--------------|----------|
| xxx.pptx | 5.2MB | [链接] | 2026-04-12 |
```

## 七、监控与告警

**监控指标**：
1. GitHub → Gitee 同步延迟
2. 知识库文件总数波动
3. 二进制文件数量/大小
4. Wiki节点访问量

**告警规则**：
- 同步延迟 > 1小时 → 飞书通知
- 文件丢失 → 立即告警
- 二进制文件超过100个 → 预警
