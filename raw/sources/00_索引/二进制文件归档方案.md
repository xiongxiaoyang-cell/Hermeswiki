## 二、归档方案

### 方案A：使用 GitHub LFS（推荐）

**优点：**
- 二进制文件单独存储，不影响仓库大小
- GitHub 原生支持
- 团队成员无需额外操作

**操作步骤：**
1. 初始化 Git LFS
```bash
git lfs install
git lfs track "*.pptx"
git lfs track "*.pdf"
git lfs track "*.docx"
git lfs track "*.xlsx"
```

2. 将二进制文件迁移到 LFS
```bash
git add *.pptx *.pdf
git commit -m "迁移二进制文件到Git LFS"
git push
```

### 方案B：云盘 + 索引（替代方案）

如果 Git LFS 配额不足，可以使用：
1. 阿里云 OSS / 腾讯云 COS
2. 在知识库中保留索引文件（MD），包含云盘链接

## 四、执行建议

### 立即执行（推荐）

1. **创建 `.gitattributes` 配置 LFS**
```
*.pptx filter=lfs diff=lfs merge=lfs -text
*.pdf filter=lfs diff=lfs merge=lfs -text
*.docx filter=lfs diff=lfs merge=lfs -text
*.xlsx filter=lfs diff=lfs merge=lfs -text
```

2. **创建二进制文件索引文档**
在每个分类下创建 `二进制文件索引.md`，记录：
- 文件名
- 大小
- 云盘链接（如有）
- 最后更新日期

### 暂缓执行

由于 GitHub LFS 有免费额度限制（1GB），建议：
- 大于 10MB 的文件迁移到云盘
- 保留小于 10MB 的文件在 LFS 中
