# 知识库命名规范指南

> 本规范旨在统一知识库文件命名，提升可读性和可维护性

---

## 一、命名格式规范

### 推荐格式

```
[分类]-[序号]-[标题].md
```

**示例：**
```
04_HR-001-员工入职流程.md
04_HR-002-薪酬核算规范.md
12_运营-001-Agent协作规范.md
```

## 二、禁止使用的格式

| 问题格式 | 说明 | 替代方案 |
|----------|------|----------|
| `20260325-xxx.md` | 日期开头 | `[分类]-[序号]-描述性标题.md` |
| `2026-03-25-xxx.md` | 长日期格式 | 同上 |
| `P1_xxx.md` | P序号开头 | `[分类]-[序号]-xxx.md` |
| `P2_xxx.md` | P序号开头 | 同上 |
| `示例_xxx.md` | 示例前缀 | 正式命名或移至demo目录 |
| `browser_xxx.md` | 浏览器生成 | 改为描述性标题 |
| `douyin_xxx.md` | 平台前缀 | `[分类]-[序号]-主题.md` |

## 三、特殊情况处理

### 3.1 任务追踪文件

**现状：** `P1_HR模块重合性分析.md`, `P1整改执行结果.md`

**建议改为：**
```
12_运营-任务-001-HR模块重合性分析.md
12_运营-任务-002-整改执行结果.md
```

### 3.2 自媒体热点文件

**现状：** `browser_hot_20260327.md`, `douyin_hot_2026-03-28.md`

**建议改为：**
```
07_内容-热点-202603-抖音出海TOC分析.md
07_内容-热点-202603-多平台热点追踪.md
```

### 3.3 选题文件

**现状：** `20260325-选题报告.md`, `20260327-选题报告.md`

**建议改为：**
```
07_内容-选题-202603-周度选题报告.md
07_内容-选题-202604-周度选题报告.md
```

### 3.4 示例/演示文件

**现状：** `示例_知识库总览_摘要.md`

**建议改为：**
```
00_索引-示例-AI摘要生成示例.md
```

## 四、执行优先级

### 高优先级（建议立即修改）
1. `P1_xxx.md` → 标准化命名
2. `示例_xxx.md` → 标准化命名
3. `20260412_xxx.md` → 标准化命名

### 中优先级（可批量处理）
4. 自媒体热点文件 → 批量重命名
5. 选题报告 → 统一格式

### 低优先级（保持现状）
- 日期格式的会议纪要（如 `2026-03-26-会议纪要.md`）
- 合同文件（如 `2026-03_ALFRED_LingKwanYew_顾问合同.md`）

---

## 五、工具支持

### 批量重命名脚本

```bash
# 查看待重命名文件
./scripts/rename_check.sh

# 执行重命名（需确认）
./scripts/rename_check.sh --execute
```

---

## 六、注意事项

1. **保持引用完整性**：重命名后需同步更新相关文档的交叉引用
2. **分批执行**：建议按分类分批执行，避免混淆
3. **保留历史版本**：旧文件移动到 `history/` 子目录归档
