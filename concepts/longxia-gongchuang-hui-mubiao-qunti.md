# 出海HR共创会目标群体画像与价值点分析

> 来源：David 发来的内部分析文档（Word格式），基于双维度分类体系（职位层级 × AI能力）。

## 一、目标群体定位

### 1.1 双维度分类体系

| 维度 | 说明 |
|------|------|
| 职位层级维度 | 普通HR从业者 / 资深HR管理者 |
| AI能力维度 | AI热忱度 + 专业深度 |

### 1.2 三类目标群体

| 群体 | 特征 | 核心定位 | 能力标签 |
|------|------|----------|----------|
| **A类** | 专业深厚 + AI薄弱 | 核心知识贡献者 | 专业积累丰富，需要AI工具辅助 |
| **B类** | 初期经验 + AI有热忱 | 问题还原者 + 框架共创者 | 有使用热情，代表使用者视角 |
| **C类** | 专业深厚 + AI有成果 | 质量验证者 + 引导者 | 双重专家，能评审和引领方向 |

## 二、群体画像详解

### 2.1 A类群体画像（专业深厚 + AI薄弱）

**典型场景：**
- 多次处理过海外用工，熟悉各国合规要求
- 积累了大量实战案例和"只能意会"的经验
- 希望经验能被结构化，不只是存在于脑子里

**画像维度：**

| 维度 | 描述 |
|------|------|
| 职位层级 | 资深HR从业者、HRM、HRD |
| 从业经验 | 5年以上出海HR专业经验 |
| AI能力 | AI薄弱——对AI工具接触较少或使用不深 |
| 核心能力 | 操作层：Checklist/FAQ/数据/模板；决策层：判断框架/踩坑案例/潜规则 |
| 参与动机 | 让经验传承、为行业贡献、与同行交流 |

### 2.2 B类群体画像（初期经验 + AI有热忱）

**典型场景：**
- 刚接触出海HR，每天花大量时间查资料
- 想找一个能帮自己快速找到答案的工具
- 愿意给产品提反馈，帮助产品做得更好

**画像维度：**

| 维度 | 描述 |
|------|------|
| 职位层级 | 普通HR从业者、专员、主管 |
| 从业经验 | 1-3年出海HR相关经验 |
| AI能力 | AI有热忱——愿意尝试、愿意学习 |
| 核心贡献 | 痛点场景、使用者视角、知识吸收路径、验证反馈 |
| 参与动机 | 获得好用工具、学习AI技能、与行业前辈交流 |

### 2.3 C类群体画像（专业深厚 + AI有成果）

**典型场景：**
- 对AI有深度使用经验，能判断AI输出的质量
- 在行业内有影响力，能为产品背书
- 见过足够多，能指出"这个不对"或"这个还不够"

**画像维度：**

| 维度 | 描述 |
|------|------|
| 职位层级 | 资深HRD、HRBP、区域HR负责人 |
| 从业经验 | 8年以上出海HR管理经验 |
| AI能力 | AI有成果——已在工作中有AI应用实践 |
| 核心贡献 | 规则验证、边界厘清、影响力背书、萃取质量评审 |
| 参与动机 | 定义行业标准、影响行业走向、把控质量 |

## 三、与职位层级框架的融合

| 群体 | 职位层级 | AI能力 | 共创会角色 |
|------|----------|--------|------------|
| B类 | 普通HR为主 | 初期经验+AI热忱 | 问题还原者、使用者视角、验证反馈 |
| A类 | 资深HR为主 | 专业深厚+AI薄弱 | 核心知识贡献者（操作层+决策层） |
| C类 | 资深HR中的专家 | 专业深厚+AI有成果 | 质量验证者、引导者、影响力背书 |

## 四、三类群体的招募配比建议（15-20人规模）

| 群体 | 建议占比 | 人数 | 角色定位 |
|------|----------|------|----------|
| A类 | 50-60% | 8-12人 | 主要知识贡献来源 |
| B类 | 25-35% | 4-7人 | 使用验证、反馈收集 |
| C类 | 10-15% | 2-3人 | 质量把关、方向引领 |

## 五、各群体对共创会的价值点

### 5.1 各群体贡献矩阵

| 群体 | 能贡献什么 | 对产品的价值 | 对其他参与者的价值 |
|------|------------|-------------|------------------|
| A类 | 操作层：Checklist、FAQ、数据、模板；决策层：判断框架、踩坑案例、潜规则 | 核心内容来源，决定产品能不能用 | 提供经验借鉴、减少同行踩坑 |
| B类 | 使用者视角的痛点、最需要的功能、验证反馈 | 产品实用性验证，决定产品好不好用 | 提问推动讨论、验证内容可理解性 |
| C类 | 规则验证、边界厘清、质量评审、影响力背书 | 内容质量把关，决定产品是否专业 | 引领方向、提供背书、提升信任感 |

### 5.2 各群体在共创会各环节的参与方式

| 环节 | A类参与方式 | B类参与方式 | C类参与方式 |
|------|-------------|-------------|-------------|
| Day1 私董会 | 分享经验、贡献判断框架 | 提问、使用者视角反馈 | 主持讨论、评审观点、方向把控 |
| Day2-3 线上采集 | 主导内容贡献：知识/Skill/数据 | 参与任务、使用产品、提交反馈 | 审核内容质量、给出修正意见 |
| Day4 总结发布 | 展示贡献成果 | 体验0.2、反馈差异感 | 总结点评、发布背书 |

## 六、HR群体对AI智能体的心态洞察

### 6.1 三大典型心态类型

| 类型 | 特征描述 | 核心诉求 | 与群体对应 |
|------|----------|----------|-----------|
| 观望试探型 | 有兴趣但怕踩坑——"这东西准不准？" | 安全感、确定性 | 主要集中在A类 |
| 积极拥抱型 | 追求效率，愿意容忍一定错误率 | 效率提升 | 主要集中在B类 |
| 抗拒焦虑型 | 担心被AI取代 | 身份确认、价值证明 | 集中在A类，需要特别处理 |

### 6.2 出海HR的独特心态特征

| 心态特征 | 具体表现 | 对共创会设计的影响 |
|----------|----------|------------------|
| 合规风险是最大顾虑 | 出海用工各国法规不同，一旦出错可能引发劳动纠纷 | 2.0版本功能定位为"国别合规速查"，直击最痛点 |
| 信任门槛高 | HR宁可用自己经验换确定性 | 需要C类背书、场景化验证 |
| 要的是"工具"不是"替代" | 帮他们快速找到正确答案，而非做决策 | 产品定位为"助手"而非"决策者" |
| 消除AI焦虑 | 担心被AI取代 | B类参与体验、A/C类引导 |

## 七、对各群体的价值点分析

### 7.1 对A类群体（专业深厚+AI薄弱）

| 阶段 | 价值点 | 具体呈现 |
|------|--------|----------|
| Day1 现场 | 经验资产化 | 将多年积累的经验结构化沉淀 |
| | 行业影响力 | 成为AI龙虾领域的贡献者 |
| | 圈子价值 | 与同行深度交流 |
| Day2-3 线上 | 贡献被记录 | Checklist/案例等被署名收录 |
| | 问题共创 | 与B类碰撞，明确什么是真正需要的 |
| Day4 现场 | 蒸馏成果 | 看到自己的经验被萃取进产品 |
| | 行业发布会 | 见证自己参与创造的产品发布 |
| 长期 | 顾问委员会席位 | 影响产品后续方向 |
| | 经济回报 | 未来Token分润优先权 |

### 7.2 对B类群体（初期经验+AI有热忱）

| 阶段 | 价值点 | 具体呈现 |
|------|--------|----------|
| Day1 现场 | 获得1.0版本工具 | 拿到可用的AI龙虾 |
| | 学习机会 | 了解AI能做什么、怎么用 |
| | 同行交流 | 认识A类前辈 |
| Day2-3 线上 | 边用边学 | 在真实任务中使用，异步支持 |
| | 被听见 | 提交的问题和反馈会被认真对待 |
| Day4 现场 | 升级到2.0版本工具 | 免费体验更强大的版本 |
| | 成就感 | 自己的反馈推动了产品改进 |
| 长期 | 持续迭代 | 产品越来越好用 |

### 7.3 对C类群体（专业深厚+AI有成果）

| 阶段 | 价值点 | 具体呈现 |
|------|--------|----------|
| Day1 现场 | 定义标准 | 用专业影响力定义行业方向 |
| | 圈子核心 | 与同level专家深度交流 |
| Day2-3 线上 | 评审角色 | 审核内容质量，给出专业意见 |
| | 定制咨询 | 在评审过程中获得个性化问题解答 |
| Day4 现场 | 质量发布 | 为2.0版本发布提供专业背书 |
| | 影响力延伸 | 成为产品的顾问和代言人 |
| 长期 | 顾问委员会 | 主导产品路线图决策 |
| | 行业地位 | 成为AI+HR领域的意见领袖 |
